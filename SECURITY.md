# Security

## Threat model

RailGraph is a self-contained graph-analytics dashboard over a committed JSON
dataset. No authentication, no user-supplied files, no database, no network calls
at runtime, no LLM, no secrets. The attack surface is essentially the Streamlit
process itself.

## What is mitigated

| Risk | Status |
|---|---|
| Secrets in git history | **Clean** — `gitleaks`: 0 findings; no `.env`, no keys anywhere |
| Dependency CVEs | **Clean** — `pip-audit`: no known vulnerabilities; versions pinned |
| Code execution / injection | **Not applicable** — no `eval`/`exec`/`subprocess`, no SQL, no user file input |
| Untrusted deserialization | **Not applicable** — data is JSON via `json.load`, not pickle |
| Data provenance | The network is a **representative/synthetic** Indian-railway topology generated by `scripts/generate_data.py` for graph-analysis demonstration — it is not claimed to be an authoritative rail map |

## What is NOT mitigated / notes

- **No authentication.** Public read-only dashboard; there is nothing to write and
  nothing private to protect.
- The dataset is a model for demonstrating centrality, k-shortest-paths, community
  detection, and resilience simulation — treat the numbers as illustrative of the
  *methods*, not as operational railway data.
